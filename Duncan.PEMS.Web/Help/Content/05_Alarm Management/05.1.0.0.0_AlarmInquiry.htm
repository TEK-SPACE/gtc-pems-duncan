Alarm Inquiry

 

To Apply Filters to an Alarm Inquiry

After selecting the Alarm Management module in the Management Pane you may filter your alarm inquiry using the following database fields.

Field Name Description
Date Range Date range to include in the inquiry
Asset Type A specific type of hardware located in the field
Asset ID A unique identifier for each discrete asset in the system.
Asset Name A unique name for each discrete asset in the system.
Alarm Status Indicates if the alarm is still active (open) or cleared (closed).
Alarm Severity Indicates if an alarm is severe, major or minor.
Operational State Indicates if the hardware is operational or non-operational.
Target Service Indicates if the status of the target service level for the asset.
Time Type Narrows the inquiry to a specific time of the week (day, weekend, etc.).
Alarm Code Identifies the specific alarm being raised.
Alarm Source Identifies where the alarm originated from (meter, gateway, etc.).
Location Type Select a location type (a geographical grouping of hardware in the field) and the data field directly to the right will display a field for that type.
Technician ID A unique identifier for each technician.

 

Viewing the Alarm Inquiry

After applying filters and refreshing the alarm inquiry, the results will display in the table at the bottom of the screen.
